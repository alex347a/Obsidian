## User story
Hvordan vil systemet kunne bruges
Kode skal kun skrives hvis det skaber værdi
Alt produktion tager udgangspunkt i dette
**As a 'Role' I can 'capability', so that 'receive benefit**
Alternativt:
**In order to 'receive benefit' as a 'role', I can 'goal/desire'**
## Characteristics of good user stories (INVEST)
- Independent: should be self-contained, so it doesn't depend on other stories.
- Negotiable: Not a contract, but a starting point conversation.
- Valuable: Must deliver value to the end user.
- Estimable: Should be able to estimate the effort required.
- Small: Should be small enough to be completed in a single iteration.
- Testable: Must be able to verify that the story is implemented correctly.
